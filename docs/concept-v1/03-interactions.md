# インタラクション設計

[01-purpose.md](./01-purpose.md) の媒体、[02-vision.md](./02-vision.md) の勾配を、StackChan ファームウェアと同席の **操作** として具体化する。

実装は `stack-chan/firmware/`（upstream）と `overlay/`（パッチ・MOD・探求用コード）。ここでは振る舞いの指針だけを書く。

---

## 原則

| 原則 | ルール |
|---|---|
| 非指示 | 初見で「こう操作して」と教えない。口頭の司会も最小 |
| 低攻撃 | 強いフィードバック・即時性・評価 UI を置かない |
| 間 | サーボ・LED・表情の変化に 300ms〜1.5s 程度の余白を入れる |
| 非線形 | 呼吸周期は完全固定にしない（小さなゆらぎ） |
| 第三焦点 | 会話エージェントではない。返答・通知・スレッドは作らない |
| 勾配 | 最初から言葉を求めない。後半は **人の口頭** に任せる |

**禁止:** 音声 Q&A、ウェイクワード、チャット UI、いいね、滞在時間の読み上げ、「よくできました」系のフィードバック、参加者全員への操作要求。

---

## セッションの勾配

同室 2〜10 人 + 机の上 1 台（複数台は選択肢）を前提とする。時間は版ごとに観察で詰める。**固定脚本にはしない。**

| 段階 | 人間側 | StackChan 側 |
|---|---|---|
| **前半** | 同席、緊張、沈黙があってよい | 共有呼吸だけ。説明・促しなし |
| **中盤** | 視線がロボットに落ちてもよい。合わせなくてよい | サーボ + LCD（+ 任意で LED）で同じリズムを続ける |
| **後半** | 言葉・価値観のすり合わせ（口頭） | テンポを変えないか、わずかに弱めるかは観察で決める |

ロボットが **後半で話題を提供したり、質問を投げたりしない**。勾配の後半は人—人。

## 層構造

Web v0.2.0 の Layer 0〜3 を、StackChan 向けに読み替える。

### Layer 0 — 呼吸（常時）

共有リズムの主軸。**電源を入れたら自動で始まる** 想定（開始ボタンを参加者に求めない）。

| 出力 | 振る舞い |
|---|---|
| サーボ（Y 軸） | 吸／吐に連動した頭の上下。角度は 5〜85° 内（[StackChan 注意事項](https://docs.m5stack.com/ja/StackChan)） |
| LCD | 呼吸リングに相当する静かな輪・表情。文字説明は置かない |
| RGB LED（任意） | 吸／吐・フェーズに連動した **弱い** 明度変化 |
| スピーカー | 原則ミュート。環境音・BGM も版まで置かない |

**初期候補パラメータ**（Web v0.2.0 から継承。観察で変える）:

| 項目 | 値 |
|---|---|
| 呼吸周期 | 10s（6 回/分） |
| 吸 / 吐 | 4s / 6s |
| ゆらぎ | 小さな非線形（instability 相当。固定しすぎない） |

X 軸（左右）は原則固定。首振りで「こっちを見ろ」と促さない。

### Layer 1 — 受動（気配）

| 要素 | 振る舞い |
|---|---|
| アイドル | まばたき程度の微動。出荷時 AI Agent のような能動的な芝居は使わない |
| 時間・フェーズ | LED または画面トーンの **ゆるやかな** 変化（昼夜相当。要検証） |
| ランダムイベント | 流れ星・波相当は **任意**。入れるなら稀で弱く |
| 沈黙 | 何も起きなくて正しい |

### Layer 2 — 自発（最小）

参加者に **必須の操作はない**。

| 操作 | 入力 | 備考 |
|---|---|---|
| 頭頂タッチ | 3 ゾーンタッチ | **任意**。触れなくて回る。触ったら微反応するかも版ごとに試す |
| リモートコントローラー | ESP-NOW（K151-R 等） | **司会・設計者側の選択肢**。テンポ微調整、一時停止など |
| 本体ボタン | 電源・リセット | 参加者向け UI ではない |

コントローラーは参加者全員に渡さない。操作の気配が存在負荷を上げないか journal で見る。

### Layer 3 — 微反応

| 対象 | 入力 | 反応 | 制限 |
|---|---|---|---|
| 頭頂 | タッチ（任意） | 表情のわずかな変化、または 1 回分の深い呼吸 | 連打不可。debounce 長め |
| IMU（本体を振る） | 加速度 | **出荷時のめまい表情は使わない** | 反応オフを既定候補 |
| 近接センサー | 手が近づく | 原則オフ。監視感を避ける | |

## ハードウェア別

[StackChan ドキュメント](https://docs.m5stack.com/ja/StackChan) の要素ごと。版ごとに ON/OFF する。

| 要素 | 既定 | 使う場合 |
|---|---|---|
| サーボ Y | **ON** | 呼吸の主軸 |
| サーボ X | OFF | 首振りは指示になりやすい |
| LCD | **ON** | 輪・表情。テキスト UI なし |
| RGB LED ×12 | 選択 | 吸吐・フェーズ・昼夜の気配 |
| タッチ | 選択 | 触れなくてよい場を優先 |
| スピーカー / マイク | OFF | AI Agent 無効 |
| カメラ | OFF | 監視・評価感を避ける |
| リモートコントローラー | 選択 | 司会者が距離を保って調整 |
| ESP-NOW（複数台） | 選択 | 同期が要る版だけ |

## 同室での置き方（初期候補）

観察で更新する。固定しない。

- テーブル中央付近。全員の視界に入る高さ
- 向き: 特定の個人を見続けない（正面固定またはゆっくりした中立）
- 2〜10 人が同室。スマホは場の主役にしない
- 開始: 電源 ON → 呼吸が始まる。口頭説明は短く、または無し

---

## リモートコントローラー（ESP-NOW）

[K151-R](https://docs.m5stack.com/ja/StackChan) 等を **選択肢** として使う場合の候補操作。全部入れる必要はない。

| 操作 | 用途 |
|---|---|
| テンポ ± | 場のリズムを司会者が微調整 |
| 一時停止 / 再開 | 後半の言葉に移る前の区切り（任意） |
| フェーズ合図 | LED 色の切替など、低解像度の段階表示 |
| 複数台同期 | 同一 Receiver ID でグループ制御 |

ジョイスティックでの連続操作は、参加者の前で派手に行わない。

---

## RGB LED

12 個（WS2812C）の **利用候補**。ON にする版と OFF の版を両方観察する。

| 用途 | 振る舞い |
|---|---|
| 吸 / 吐 | 明度または色相のゆっくりした往復 |
| フェーズ | 前半／中盤／後半の低解像度な色の shift |
| 時間 | v0.2.0 の昼夜に相当するトーン（要検証） |

避ける: 点滅、レインボー、ゲーム的フィードバック、音量連動。

---

## 言葉（人間側）

StackChan には **入力 UI を載せない**。

| 段階 | ルール |
|---|---|
| 前半〜中盤 | ロボット経由の入力なし。沈黙 OK |
| 後半 | 口頭でのすり合わせ。チャット・WordBar 型は使わない |
| 記録 | ファームウェアやサーバーに会話を保存しない（探求版） |

言葉を「後半に回す」こと自体も、勾配が自然に生まれたか観察してから固定する。

## Web v0.2.0 からの対応

| Web（v0.2.0） | StackChan（v1.0.0 以降） |
|---|---|
| `useBreathEngine` | サーボ + LCD の呼吸ループ |
| `useDayCycle` | LED / 画面トーン（任意） |
| Orbs / 星空 | LED の弱い気配（人数表示ではない） |
| TimeSeekBar | **なし**（司会者コントローラーで代替候補） |
| Help / `?` | 口頭最小、または紙 1 枚まで（要検証） |
| Ripple / click | タッチ微反応（任意）またはなし |

## 版ごとの取捨

journal とセットで、次のような **版ラベル** を付けて観察してもよい。

| ラベル例 | 内容 |
|---|---|
| `breath-only` | Layer 0 のみ（サーボ + LCD） |
| `+led` | RGB LED 追加 |
| `+touch` | 頭頂タッチ微反応 |
| `+remote` | 司会者コントローラー |
| `+multi` | ESP-NOW 複数台 |

---

## 観察で見る副作用

- 操作を求められた感覚
- ロボットが **第三焦点** か、監視装置か
- LED / コントローラーが勾配を助けたか、演出になったか
- 呼吸だけの「無」と、輪郭が戻る瞬間
- 後半の口頭すり合わせは自然に出たか
