# インタラクション設計

## 原則

1. **呼吸を土台にする。** すべての表現は呼吸の上に乗り、呼吸を壊さない
2. **沈黙を既定にする。** 動き、光、音、反応は稀で弱くする
3. **非指示にする。** 注目、返答、正しい行動を要求しない
4. **受動を優先する。** 音声内容を録音・認識・送信せず、扱う場合もレベルと特徴量に限る
5. **即時性を避ける。** 反射以外の反応には間を置き、戻りをゆっくりにする
6. **固定周期を避ける。** 自発挙動は非線形な間隔にする
7. **能力に正直である。** センサーで知覚できない対象を見ている演技や、理解している演技をしない
8. **評価しない。** 人の行動への報酬・罰・拒絶に見える表現を避ける

## 表現の層

| 層 | 役割 | 例 |
|---|---|---|
| 呼吸 | 常に続く土台 | 目の脈動、頭の上下、LED の緩やかな変化 |
| 気配 | 自己完結した生存感 | まばたき、視線の漂い、深呼吸、まれな murmur |
| 微反応 | センサー入力への弱い反射 | 音への一瞥、startle、タッチへの短い反応 |
| ムード | 長い時間尺度の連続変調 | 覚醒・沈静、表現量、姿勢、色相の変化 |

上位の層ほど頻度と強さを慎重に扱う。すべてを有効にすることを目標にしない。

## ハードウェアの扱い

| 要素 | 方針 |
|---|---|
| LCD | 口のない目を中心にし、説明テキストや社会的 UI を置かない |
| サーボ | ゆっくりした姿勢と視線追従に限り、駆動音が場を壊す場合は抑える |
| LED | 低輝度の呼吸と内部状態に使い、点滅・レインボー・ゲーム的フィードバックを避ける |
| スピーカー | 短く柔らかい音だけを稀に使い、無音にできる状態を保つ |
| マイク | 音声内容を扱わず、音量・特徴量・破裂音の左右推定だけに使う |
| タッチ | 任意の微反応に留め、操作を必須にしない |
| カメラ | 監視・評価感を避けるため使わない |
| リモコン・複数台 | 必要性を観察で確認するまで主経路にしない |

## 人間側の言葉

StackChan に会話入力 UI を載せず、会話を保存しない。言葉は同席する人の間で口頭により生まれる余地を残す。

## 観察する副作用

- 操作や反応を求められた感覚がないか
- 第三の焦点ではなく監視装置に見えないか
- 動き、LED、音、サーボ音が注意を奪わないか
- 生き物らしさが芝居や誘いに変わっていないか
- 沈黙の中でも死んでいるように見えず、静かな輪郭を保てるか
- 後半の言葉が無理なく生まれたか
