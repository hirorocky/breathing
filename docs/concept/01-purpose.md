# 探求の目的

**社会の中で、演じなくていい時間を、同じ部屋で試す。**

存在負荷が高い私たちが、StackChan を第三の焦点として同じ部屋に置き、高い存在負荷から低い状態へゆるやかに向かえる時間を探求する。おおよそ 2〜10 人での同席を想定する。

## 問題意識

同じ空間にいるとき、次の負荷が生まれやすい。

- 何を話すか決め続ける
- 沈黙を埋める
- 印象を管理し、正しく自己提示する
- 相手の反応を読み続ける

この負荷を孤立によって避けるのではなく、**人との関係の中で下げられるか**を試す。

## スタンス

- 人は要る。StackChan は人の代わりではなく、人—人の間に置く共有リズムと第三の焦点である
- 会話や反応を要求しない。沈黙、曖昧さ、未完成な状態を許す
- 出荷時の AI Agent、質問応答、音声ウェイクワードは使わない
- 読める社会的 UI、評価指標、常時入力によって他者感を合成しない
- 完成形を先に決めず、観察から設計を修正する

## 仮説

- 最初から言葉を求めず、呼吸のような共有リズムを先に置くと、存在負荷に勾配が生まれる
- 小さな身体が静かに同席すると、人間同士の視線や沈黙の圧力を第三の焦点へ逃がせる
- 稀で弱い気配は、注意を奪わずに「そこに生きている」感覚を保てる
- 言葉や価値観のすり合わせは、場の後半に口頭で自然に生まれる余地を残す

## 探求のループ

| 段階 | やること |
|---|---|
| 修正 | StackChan の振る舞い、間、部屋での置き方を変える |
| 観察 | 同席して起きたことを見る |
| 気づき | 存在負荷と設計の副作用を言葉にする |
| 次の修正 | 気づきを設計判断へ戻す |

観察対象を固定するときは git tag を付け、ユーザーの観察は [journal](../journal/) に残す。

## 成功条件

人数や滞在時間ではなく、次の質感が生まれたかで判断する。

- しばらく演じなくていられた
- 相手がそこにいる感覚と、自分の輪郭を同時に保てた
- 沈黙や言葉が無理なく同じ場にあった
- 操作ではなく関係の質として、存在コストが下がった
