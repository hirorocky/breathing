# 実験の目的

## 何をしたいか

「存在負荷を下げる空気」を、実際に動く Web 上に置く。

人が過剰に補正したり演じたりせず、比較的そのまま存在できる余白を持った場を増やしたい。居場所づくりや「優しい世界」ではなく、次を試す小さな実験である。

- 補正量を下げる
- 存在コストを下げる
- 未完成な状態を許容する
- 過剰な自己演出を減らす

## 問題意識

多くのオンライン空間では、次が起きやすい。

- キャラクター化
- 承認獲得競争
- 即時反応の圧力
- 正しいコミュニケーションへの適応

その結果、思考途中の状態・曖昧さ・試行錯誤・本来のテンポが失われていく。

## 仮説

存在負荷の低い空間には、次の特徴があるのではないか。

- 少し黙っていてもよい
- 解像度が合っていなくてもよい
- 即座に価値を出さなくてもよい
- 未完成な言葉を置いておける
- キャラクターを固定されない

## 検証したいこと

- どの設計が存在負荷を下げるか
- どの機能が補正や自己演出を生むか
- 未完成な言葉を残せる場は成立するか
- 承認指標を減らしても場は維持できるか
- 人はいつ「存在しやすい」と感じるか

## 成功条件

利用者数や滞在時間では測らない。次の体験が生まれること。

- 少し補正せずにいられた
- 未完成なまま表現できた
- 存在コストが下がった

KPI ダッシュボードや成長指標は載せない。空気を観察しながら、小さく改善を続ける。

## オンライン化の位置づけ

利用者を増やすことが目的ではない。不特定多数が同じ空間に重なったとき、設計が仮説を支えるかを観察するためのインフラを用意する。

オンラインで意図的に入れないもの:

| パターン | 負荷が上がる理由 |
|---|---|
| キャラクター化 | 一貫した自己演出が要求される |
| 承認獲得競争 | いいね・ランキングが補正を促す |
| 即時反応の圧力 | 投稿直後のフィードバック期待 |
| 正しいコミュニケーションへの適応 | スレッド・返信・既読 |
