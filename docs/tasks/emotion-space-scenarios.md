# 感情 2 次元空間とシナリオ 20(ドラフト — 取捨選択用)

v1.1.0 Phase 3c の先(または v1.2.0)に向けたブレインストーミング。**concept-v1 の制約(稀・弱・受動)は意図的に外して書いている** — 採用時に絞る前提。ELEGNT の設計台帳(`elegnt-expression-design.md`)の 4 語彙で行動を記述する。

## 空間の定義(Russell の感情円環)

- **X = 快‐不快(valence)** ∈ [−1, +1]。担当アセット: **LED の色相・音色**(柔らかい/短く低い)・topAngle(険しさ)・視線の向けやすさ(向ける = 受容 / 外す = 拒否)。~~botArc(smile-eyes)~~ は 2026-07-08 廃止 — 「笑ったときの目に違和感。喜びは目ではなく LED・声色・動きで表現」(ユーザー確定)
- **Y = 覚醒‐沈静(arousal)** ∈ [−1, +1]。担当アセット: 呼吸周期・脈動深さ・サッカード頻度と速度・目の大きさと高さ(疑似プロクセミクス: 大きく上 = 接近、小さく下 = 退行)・まばたきの速さ

## 力学

1. **イベントがベクトルを加算**: 拍手/loud → arousal+(valence は文脈)、touch → valence+、voice 継続 → 中覚醒安定、silence 継続 → arousal−、時刻 → arousal の上限/ベースラインを変調
2. **指数回帰**: 常時ゆっくりベースライン(やや快 +0.2・中覚醒 0)へ戻る(時定数 数分)。ポアソン的な微ノイズで完全静止しない(非線形原則)
3. **位置が反応スタイルを変調**: 同じトリガでも現在位置で 速度・振幅・確率・回復時間・音色 が変わる(これが「生き物の連続性」の正体)

## シナリオ 20

象限: Q1 = 快×覚醒(はしゃぎ)、Q2 = 不快×覚醒(警戒)、Q3 = 不快×沈静(しょんぼり)、Q4 = 快×沈静(まどろみ)。

| 1 | ごきげん | Q1 中庸に滞在 | valence 高いとき全般 | サッカード活発・目が丸く上向き・murmur が明るい音色に。反応は速く大きい(attention のゲイン↑) |
| 2 | はしゃぎ疲れ | Q1 端 → Q4 へ滑落 | 高覚醒が数分継続 | 興奮は維持できない — 動きが徐々にゆっくりに(長い follow-through)。生き物の減衰 |

## 実装スケッチ(採用が決まったら)

- `emotion.js`: 状態 `{v, a}` + イベント購読(onMicEvent・touch・時刻)+ 指数回帰 Timer + `getEmotion()`
- 変調の配線: liveliness(サッカード頻度・murmur 頻度)・reactions(不応期・振幅・cry 確率・回復時間)・breath(周期)・eye-cozmo(形状プリセット補間)が `getEmotion()` を読む
- 表情は B 案の変形パラメータ(topLid/topAngle/scale/asym — botArc は廃止)を空間座標から連続補間(プリセット間のモーフ)— 記号は使わない(Cozmo 文法)
- すべての係数は `GET/PUT /emotion/params` でライブ調整(既存パターン)

## 注意(採用時に concept-v1 と照合する点)

- #5/#6/#19 の「怒り・拒否」系は評価的フィードバック(人の行動への罰)に見えないか — 場への反応であって人への評価ではない、の線引きが必要
- #10 の自己刺激・#18 の回復儀式は「非指示」と相性が良いが、頻度を誤ると誘いになる
- #20 は声の内容ではなくレベル変動の統計だけを使う(受動原則の維持)
