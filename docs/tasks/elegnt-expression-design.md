# ELEGNT × breathing — 表現設計

Hu et al. (2025), [“ELEGNT: Expressive and Functional Movement Design for Non-anthropomorphic Robot”](https://arxiv.org/abs/2501.12493) の語彙を、[breathing の指針](../concept/)に合わせて使う。

## 採用する考え方

ELEGNT は動作を、目的へ到達する機能的な動きと、内部状態を伝える表現的な動きに分ける。内部状態の語彙は次の4つである。

- **Intention**: 行動前の予備動作
- **Attention**: 対象への注意や一瞥
- **Attitude**: 反射、受容、ためらいなどの構え
- **Emotion**: 長い時間尺度のムード

breathing は engagement を最大化しない。語彙と動作文法だけを借り、表現を「関わりの要求」ではなく「生きている気配」に使う。

## breathing への翻訳

- 機能的な土台は、途切れず呼吸することである
- 表現量は一定にせず、静かな場ではほぼゼロまで下げる
- センサーで知覚できない対象を見ている演技をしない
- 常時動かず、ポアソン的な間隔と長い無反応時間を保つ
- モーター音を表現の副作用として評価する
- 音、目、LED、姿勢のタイミングを揃える
- 人への報酬・罰・拒絶に見える情動表現を避ける

## 語彙と表現

### Attention — 一瞥

音を検出してから短い間を置き、目を向け、保持し、ゆっくり中央へ戻る。破裂音だけはステレオマイクの左右推定を使える。連続音では方向を演じない。凝視や話者追跡はしない。

### Intention — 予備動作

深呼吸の直前に小さく沈む、目が先に動いて頭が遅れて追う、といった staging を使う。予告は知覚できる最小量にする。

### Attitude — 反射と受容

startle は立ち上がりを速く、回復を遅くする。連発を防ぐ不応期を置き、音は確率的に鳴らす。タッチには短い音や暖色の波を返せるが、返事や評価に見せない。

### Emotion — 場のムード

valence と arousal を連続値として扱い、呼吸、視線、瞼、LED、姿勢、反応の速度と回復をまとめて変調する。離散的な顔文字や記号表現は使わない。

## 時間的な規則

1. 反射以外は slow-in / slow-out にする
2. 反射以外の反応には間を置く
3. attack より release を遅くする
4. 音を出すときは顔や身体にも同じ反応を載せる
5. 固定周期で繰り返さない

## 評価のガードレール

| リスク | 確認すること |
|---|---|
| 表現過多 | 静かな作業や沈黙を邪魔しないか |
| 能力の誇張 | 実際には知覚していない対象への演技になっていないか |
| 監視感 | 一瞥や凝視が人を追跡しているように見えないか |
| モーター音 | 動きの意味より駆動音が勝っていないか |
| 誘い | 自発挙動が「こちらを見て」と要求していないか |
| 評価 | 不機嫌、拒否、喜びが人への報酬・罰に見えないか |
| 不整合 | 顔、音、LED、姿勢の変化が同じ内部状態を示しているか |
