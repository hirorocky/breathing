# ELEGNT × breathing — StackChan の表現設計

参照: Hu, Huang, Sivapurapu, Zhang (Apple, 2025) "ELEGNT: Expressive and Functional Movement Design for Non-anthropomorphic Robot" ([arXiv:2501.12493](https://arxiv.org/abs/2501.12493))。ランプ型ロボット（Luxo Jr. 系、6 自由度アーム + 光 + 音声）で、動作の効用を **機能 F(τ) + γ×表現 E(τ)** に分解し、表現的動作がユーザーの知覚（engagement・生き物らしさ・キャラクター性）を大きく高めることを N=21 の within-subject 実験で示した研究。

このドキュメントは ELEGNT の設計語彙を breathing の理念（`docs/concept-v1/`）と衝突しない形で StackChan のアセットに翻訳する。**Loop C / Phase 3c（顔・音・サーボを組み合わせた総合的な動き）の設計台帳**として使う。

---

## 1. ELEGNT の要点（内部化した骨格）

- **Function-driven movement (F)**: 初期状態から目標状態への効率的な到達。**Expression-driven movement (E)**: 同じ目標でも、意図・注意・態度・感情が伝わる**経路**を選ぶ。統合は `max F(τ) + γE(τ)`
- 表現が伝える内部状態は 4 語彙:
  - **Intention（意図）** — 行動の前にターゲットへ頭を向ける等、次の行動の予告
  - **Attention（注意）** — 話者や対象への視線、joint attention
  - **Attitude（態度）** — 頷き・ためらいの間・限界時に身体を伸ばす等、反応の構え
  - **Emotion（情動）** — 軽く弾む=喜び、ゆっくり=リラックス、頭を下げる=悲しみ
- 設計プリミティブは **kinesics**（空間的: 姿勢・ジェスチャ / 時間的: 速度・間・加速度）と **proxemics**（対象への接近・距離・向き）
- 主な発見:
  - 表現は **social-oriented な文脈で効き、function-oriented（人がタスクに集中している）文脈では効かない**（むしろ予測不能で邪魔）
  - **能力にない表現は fake に見える**（カメラが無いのに「見る」演技をする等）
  - **常時動き続けると逆に「注意散漫」に見える**
  - **モーター音は不快要因**
  - **プロアクティブな介入は自律性の侵害として嫌われる**（表現性はその受容を和らげる側）
  - **マルチモーダルの不整合**（声と動きのズレ等）は台無しにする

## 2. breathing への翻訳 — どこが同じで、どこが逆か

**同じところ**: ELEGNT の「逆効果」リストは concept-v1 の原則とほぼ 1:1 で対応する。非線形（固定周期禁止）⇔ 常時動作の嘘くささ、非指示・第三焦点 ⇔ プロアクティブ介入への抵抗、サーボ慎重論 ⇔ モーター音の不快。私たちの直感に外部の実証がついた。

**逆のところ**: ELEGNT は engagement の最大化を目指す。breathing は目指さない（第三焦点 — こっちを見ろと要求しない）。だから **ELEGNT の文法は借りるが、目的関数は借りない**。表現は「関わりたさ」ではなく「生き物としての気配」に奉仕する。

**breathing 固有の再解釈**:

1. **F = 呼吸（Layer 0、不可侵）**。StackChan にはタスクがないので、ELEGNT の F に相当するのは「呼吸し続けること」そのもの。すべての表現 E は**呼吸の上に乗る**（呼吸を壊す表現は書かない）。深呼吸は「F を通して E を表現する」模範例
2. **γ は定数ではなく γ(場)**。ELEGNT の social/function 文脈の使い分けを、マイクの部屋状態にマップする: **声がある部屋（social）→ γ をやや上げる / 沈黙が続く部屋（人が集中 or 不在 = function 側）→ γ をほぼゼロに**。「沈黙が正しい」の定量版
3. **能力の正直さ**: カメラは無い。音源方向もまだ推定していない。だから「特定の何かを見る」演技はしない — 現在のランダムサッカードは「内的な生」の表現であって知覚の演技ではないので正直。声に反応する一瞥はマイクがあるので正直。**方向つきの一瞥は、ES7210 のステレオ 2ch から左右推定を実装できたときに初めて解禁**（それまでは方向を持たせない）

## 3. 内部状態 4 語彙 × StackChan アセットのマップ

アセット: 目（脈動・まばたき・視線・変形プリセット topLid/topAngle/botArc/scale/asym）/ 呼吸サイクル / cry（murmur・startle・touch、毎回ゆらぎ）/ マイクイベント（loud・clap・voice・silence — 3b）/ タッチ / LED 12 連（未使用）/ サーボ ×2（現在 OFF）。

### Attention（注意）— 「一瞥」

- **トリガ**: 沈黙からの voice 立ち上がり（3b イベント）
- **形**: 間（300〜800ms）→ 視線がわずかに動く（方向なし・正面〜ランダム）→ 1〜2 秒保持 → ゆっくり戻る
- **禁止**: 凝視の維持（監視感）、話者追跡の演技（能力にない）
- **将来**: ステレオ左右推定ができたら初めて「音のした側」へ

### Intention（意図）— 「予備動作」

- 深呼吸の前: 0.3 秒だけ静止（その間サッカード抑制）+ 瞼がわずかに下がる — 「吸い込む前の溜め」
- murmur の前: 目がほんのわずかに縮む（息を吸う気配）→ 鳴く
- サーボ時代: **目が先、頭が後**（150〜300ms 遅れ）— アニメーションの staging。逆順は機械に見える

### Attitude（態度）— 「反射と受容」

- **startle**（loud/clap → 反射。「間」の唯一の例外）: 即時にまばたき + 目が縮む + 呼吸一拍停止 → **回復はゆっくり 2〜4 秒**（follow-through。立ち上がり速く・戻り遅くが生き物の減衰）。鳴くのは確率 10〜20%（毎回鳴くと安っぽい）。不応期は長く（連発しない）
- **touch**（頭頂）: 下弧が上がる柔らかい目（botArc、Cozmo の smile-eyes）1〜2 秒 + まれに touch cry。返事ではなく受容
- hesitation（ためらい）は当面使いどころがない — 保留

### Emotion（情動）— 「場のムード」（長時間尺度）

- **沈静**（silence が N 分継続）: 呼気をわずかに伸ばす（〜10%）・瞼 −10%・サッカード頻度減・murmur 頻度減。**評価ではなく同調**（静かな部屋と一緒に静かになる）
- **覚醒 baseline**（声のある部屋）: 現在の既定パラメータ
- **うたた寝**（深夜帯 + 長い沈黙）: 表情プリセット「眠気」+ 呼吸さらにゆっくり。どんな音でも startle ではなく「ゆっくり目が開く」で戻る（寝起きに驚かない）
- **禁止**: 評価的フィードバック（人の行動への報酬/罰として情動を使わない — concept の禁止事項そのまま）

## 4. 時間的質感の規則（kinesics temporal — 全表現に共通）

1. **slow-in / slow-out**: 反射（startle）以外のすべての動きはイーズをかける
2. **間**: 反射以外の反応は 300ms〜1.5s 遅らせる（concept 原則 = ELEGNT の deliberation pause）
3. **attack 速く・release 遅く**: どの表現も「戻り」は「行き」より必ず遅い（follow-through）。即座のリセットは機械
4. **secondary action / マルチモーダル整合**: 鳴くときは必ず顔も微変化する（音だけだと「スピーカー」に見え、生き物に見えない）。逆に大きな表情変化に音は必須ではない
5. **ポアソン的間隔**（既存原則）: 固定周期は「作り物」— ELEGNT でも常時動作は逆効果と実証済み

## 5. 疑似プロクセミクス（移動できない身体の距離表現）

StackChan は動けないので、proxemics は**目のサイズ・位置**で代替する（Cozmo と同じ手法）:

- 接近（関心）= 目がわずかに大きく・上がる
- 退行（沈静・眠気）= 目がわずかに小さく・下がる
- 現在の変形プリセット軸（scale / centerY 相当）で実装可能。振れ幅は数 % に留める（低攻撃）

## 6. Phase 3c の実装順（一つずつ、各々がユーザー FB ループ）

| # | 表現 | 語彙 | 使うアセット | 依存 |
|---|---|---|---|---|
| 1 | startle | Attitude | まばたき + 脈動停止 + 確率 cry + 回復カーブ | 3b loud/clap |
| 2 | 一瞥 | Attention | 視線 + 間 + 戻り | 3b voice |
| 3 | 沈静/覚醒の勾配 γ(場) | Emotion | liveliness パラメータの状態別セット | 3b silence |

## 7. ガードレール（ELEGNT の実証 → breathing の規則）

| ELEGNT の発見 | breathing での規則 |
|---|---|
| function 文脈では表現が邪魔 | 沈黙の部屋（人が集中）では γ ≈ 0。表現は声のある部屋でだけ増える |
| 能力にない表現は fake | 方向つき視線はステレオ推定の実装まで禁止。カメラ的演技は永久に禁止 |
| 常時動作は注意散漫に見える | ポアソン的・まれ・「沈黙が正しい」（既存原則の維持） |
| モーター音は不快 | サーボ解禁は駆動音評価とセット。鳴らすなら動きの意味が音に勝るときだけ |
| プロアクティブ介入への抵抗 | 非指示の維持。表現はすべて受動（反応）か自己完結（気配） |
| マルチモーダル不整合 | 音と顔は必ず同時に設計する（3c の各表現は顔・音・（将来）サーボの束で定義） |
