# 機能仕様

## 気配（同時接続数）

**表示:** 「いま、N人が居合わせている」+ `PresenceDots`

| 項目 | 方針 |
|---|---|
| データ | 接続セッション数（実数） |
| 更新 | 緩い更新（polling 既定 60s） |
| 非表示 | 名前、アイコン、一覧、ランキング |

0 人でも場は成立する。人数は盛り上がり指標にしない。

`apiMode === "online"` のときだけ実数。それ以外は演出 fallback。

## 言葉

**操作:** 任意キー → WordBar → Enter

| 層 | 方針 |
|---|---|
| 画面 | 自分が書いた言葉だけ `DriftField` に漂う（最大 40） |
| サーバー | POST で保存。利用者向け GET なし |
| 実装者 | `GET /api/admin/words` または D1 で閲覧 |

- 1〜24 文字、URL 禁止
- 投稿成功時も祝杯 UI なし。静かに漂うだけ
- リロードでローカルの言葉は消えてよい

## orb

`NEXT_PUBLIC_ONLINE=1` かつ `NEXT_PUBLIC_ORB_LINK≠0` のとき presence に連動。

| 項目 | 方針 |
|---|---|
| 表示数 | presence を 1〜7 にクランプ（最後が `you`） |
| 変化 | 1.2s 刻みのゆっくり追従 |
| 座標 | `sessionSeed` 由来。共有しない |

## help と開示

help は許可の文章。アンケートや実験説明ではない。

**三層の開示**（先に許可を読ませ、最後に事実を置く）:

| 層 | 置き場所 |
|---|---|
| 常時 | 左下「この場について」→ `/privacy` |
| help | `?` 内の `help-disclosure` |
| 詳細 | `/privacy` |

**使う語:** 他の利用者には見えない / 場をつくっている人 / あとから読むことがある / 個人を特定しない

**避ける語:** 監視、ログ収集、セッション ID、匿名化処理

**やらない開示:** 初回同意モーダル、投稿前後の注意ダイアログ、常時の「録画中」表示

WordBar には開示を足さない。知りたい人は `?` か「この場について」へ。

### help-disclosure（確定文案）

```
この場は、存在の負荷を少し下げられるかを試す小さな試みです。
置いた言葉は他の利用者には見えません。場をつくっている人が、あとから読むことがあります。
名前やプロフィールはありません。利用のおおよその様子だけを、個人を特定しない形で見ています。
```

### `/privacy` に書く項目

1. この場について（小さな実験）
2. 他の利用者に見えるもの（人数のみ）
3. 場をつくっている人に見えるもの（言葉の文面、おおよその利用様子）
4. 目的（存在負荷を下げる設計の試行）
5. 第三者提供なし
6. 保管（古いものから削除）
7. 問い合わせ先

## クライアント同期

| イベント | 方針 |
|---|---|
| 言葉を置く | ローカル即反映 → 裏 POST。失敗時は最大 8 件まで再送 |
| 他者の言葉 | 取得・表示しない |
| presence | `GET /api/presence` を 60s 間隔で polling |
| `static_only` 受信後 | そのセッション中は polling・POST を停止 → fallback |
