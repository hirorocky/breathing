# 機能仕様

## 時間サイクル

| 項目 | 方針 |
|---|---|
| 周期 | 実時間 6 分でサービス内 24 時間（`?debug=1` 時は 30 秒） |
| 同期 | `epochSec % period / period`。全訪問者で同じ phase |
| 表示 | phase × 24h を `0:00`〜`24:00` として下部シークバーに表示 |
| 背景 | oklch キーフレーム補間で `--bg-*` / `--ink-*` / `--ember-*` を更新 |
| 操作 | シークバーは常時ドラッグ可。`phaseOffset` はそのタブだけ。ドラッグ後も実時間で進む。リロードで同期に戻る |
| 非対象 | イベント頻度・orb 数は変更しない |

## orb

| 項目 | 方針 |
|---|---|
| 表示数 | `CONFIG.orbCount`（既定 5、最後が `you`） |
| 座標 | `sessionSeed` 由来。共有しない |

## help と開示

help は許可の文章。アンケートや実験説明ではない。

**三層の開示**（先に許可を読ませ、最後に事実を置く）:

| 層 | 置き場所 |
|---|---|
| 常時 | 左下「この場について」→ `/privacy` |
| help | `?` 内の `help-disclosure` |
| 詳細 | `/privacy` |

**使う語:** 場をつくっている人 / 個人を特定しない / 何もしなくていい

**避ける語:** 監視、ログ収集、セッション ID、匿名化処理

**やらない開示:** 初回同意モーダル、投稿前後の注意ダイアログ、常時の「録画中」表示

### help-disclosure（確定文案）

```
この場は、存在の負荷を少し下げられるかを試す小さな試みです。
名前やプロフィールはありません。利用のおおよその様子だけを、個人を特定しない形で見ています。
```

### `/privacy` に書く項目

1. この場について（小さな実験）
2. 他の利用者に見えるもの（呼吸の形のみ）
3. 場をつくっている人に見えるもの（おおよその利用様子）
4. 目的（存在負荷を下げる設計の試行）
5. 第三者提供なし
6. 保管（古いものから削除）
7. 問い合わせ先
