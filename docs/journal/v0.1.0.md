# v0.1.0

- **タグ**: `v0.1.0`（journal 確定後に付与）
- **本番**: https://breathing-4k7.pages.dev
- **観察期間**: 2026-06-05 〜 2026-06-09
- **位置づけ**: オフライン試作から本番公開、下部 UI 調整、concept / journal 運用開始まで

## この版の位置づけ

v0.1.0 は「動く探求環境が成立した」版である。

呼吸する中心の形、漂う言葉、人数の気配、ランダムイベント、Cloudflare Pages / Worker / D1 による本番公開までを含む。ここで初めて、ローカルの試作ではなく、実際に滞在して観察できる場所になった。

同時に、concept を「存在しやすい空間の探求」として整理し、**修正 → 観察 → 気づき → 次のバージョン** のループを journal に残す運用を始めた。

## この版で試したこと

### 呼吸する場

- 中心の形（ring / glow / core）が呼吸する
- 背景がゆっくり揺れる
- ポインタ・クリックに微反応する
- 流れ星・風・波のランダムイベントが起きる

### 言葉

- 自分が置いた言葉だけが画面に漂う
- 返事・いいね・通知はない
- 版の終わり時点では WordBar を常時表示
- プレースホルダーはなし。送信ボタンは文字があるときだけ出る

### 気配とオンライン

- 人数の気配を polling で表示する
- 言葉は裏で送信するが、利用者向け一覧はない
- セッション滞在を記録し、Cron で確定・削除する
- Cloudflare Pages + Worker + D1 で本番公開した

### 開示と運用

- 説明は `?` と `/privacy` に集約する
- help は使い方ではなく、許可の文章にする
- git tag と journal で、版と気づきを対応づける

## 観察

### 呼吸

- **呼吸が主役のまま滞在できる。** 操作を覚えなくても、中央の収縮を見ているだけで「そこにいる」感覚が立つ
- **単語を入力する作業は、この空間に必要ないかもしれない。** 呼吸に合わせた収縮を見るだけで足りる
- **円だけの暗い空間は少し寂しい。** 静けさは保たれているが、孤独感との境目がある

### 言葉

- **言葉に承認回路がないのはよい。** 自分の画面に漂うだけで、返事・いいね・通知がない。置いたあと静か
- **WordBar の開閉は儀式化していた。** 言葉を置く前に「場に入る」一手間があり、存在より操作が前に出た
- **常時表示にも重みがある。** 入力欄が常に見えると、呼吸より「書く場所」が前に出る瞬間がある
- **プレースホルダーを外すと静かになる。** ただし、初見で「ここに書ける」と気づくまで時間がかかる可能性がある

### 気配

- **人数は気配で止まっている。** 誰がいるかは分からないので、社交圧は弱い
- **「いま、N人が居合わせている」という文字列は寂しさを埋めない。** 人数を知っても、別の存在を感じることにはつながらなかった
- **別の存在を意識するギミックが必要そう。** ただし、それはオンライン上の人である必要はない

### 運用

- **本番が動く。** 探求の対象がローカルだけでなく、不特定の重なりも観察できる状態になった
- **表現とインフラを同じループで回せる。** SVG の表現、D1、Cron、デプロイまで含めて、修正して観察できる
- **数値 KPI を載せない方針は守れている。** 代わりに、journal で質感を言語化する必要がある

## 気づき

### 空間について

1. **存在負荷は「機能の有無」より「手順の本数」に効く。** WordBar を開く一手が、置く行為全体を重くしていた
2. **言葉を置くことは、存在の主役ではないかもしれない。** v0.1.0 では痕跡として言葉を試したが、観察の中心は呼吸の収縮に戻れる
3. **静かな場ほど、常に見える UI は存在感を持つ。** 入力欄は入口にもなるが、空気より操作を前に出すことがある
4. **寂しさは「情報の不足」ではなく「他者感の質」で決まる。** カウントやラベルでは埋まらない。何かがそこにある、という感覚が別の形で要る
5. **他者はオンラインの人である必要はない。** 同時接続という事実だけでは不十分。気配、リズム、環境の変化など、別種の存在を検討できる
6. **朝から夜、また朝を短時間で体感できると、場の温度が変わるかもしれない。** 暗さだけではなく、時間の通過そのものが明るさや落ち着きを生む可能性がある

### 探求の進め方について

7. **非指示は信頼の前提になる。** 教えないことで「正しい使い方」への適応圧を下げている。代わりに、許可（help）と痕跡で場のルールを示す
8. **オンライン化は人数のためではなく、重なりの観察のため。** 利用者を増やす装置ではなく、設計が複数の存在とどう両立するかを確かめるインフラである
9. **「完成した場」より「回るループ」が核になりうる。** 答えを先に決めず、版を切って観察し、気づきを次へ渡すこと自体が探求になる
10. **git のタグと journal をセットにすると、判断根拠が残る。** コードだけでは空気の判断が失われる。気づきは実装と別に固定したい
