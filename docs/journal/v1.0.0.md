# v1.0.0

- **タグ**: `v1.0.0`
- **本番**: 実機（K151-R / StackChan、`overlay/mods/breath`）
- **観察期間**: 2026-07-05 〜 2026-07-05
- **位置づけ**: Web から StackChan へ媒体を移し、Layer 0（呼吸のみ）を机の上に置いた初版

## この版の位置づけ

v1.0.0 は、v0.2.0 の問い——同室・第三焦点・画面を介さない容器——に向けて、**StackChan で Layer 0 だけ** を試した版である。

v0.2.0 までの Web 実装は git 履歴に残し、探求の主経路を **Moddable/JS ファームウェア + 同席観察** に移した。完成形の同室セッションまでは至らないが、「電源を入れると説明なく呼吸が始まる」最小の身体を実機に載せ、そこから観察を回す起点とする。

## この版で試したこと

### 媒体・環境

- `stack-chan/` サブモジュール + `overlay/`（パッチ・MOD・ドキュメント）で K151-R 向け開発環境を整えた
- `overlay/mods/breath/` を書き込み、実機で動作確認した

### Layer 0 — 呼吸

- 吸 4 秒 / 吐 6 秒（小さなゆらぎ）
- 電源投入 ~0.5 秒後に自動開始。開始ボタン・タッチ・説明なし
- **最終形**: 呼吸は **LCD のみ**（口の開閉 + デフォルト顔の breath motion）。サーボは **停止**（`setTorque(false)`、頭は動かさない）
- 途中版ではサーボ Y 軸も吸／吐に連動させたが、観察後に顔のみへ変更した

### 見た目

- 黒背景（`secondary`）・白パーツ（`primary`）
- `NEUTRAL`、文字・吹き出し・AI Agent なし
- LED・音声オフ

### 意図的に入れなかったもの

- Layer 1 以降（昼夜・流れ星・タッチ）
- リモートコントローラー、ESP-NOW、会話・Q&A
- スモークテスト的な派手な LED 演出

## 観察

### サーボ

- **サーボで動くと大きめな音がなる。気が散る。** 呼吸を共有するより先に、機械の気配が立つ

### 媒体（StackChan）

- **人間ではないが、Web のように人工的すぎるものでもない。** 机の上の小さな身体は、画面越しの Web とも、人の代わりとも違う質感がある（同室観察前の一人での印象）

### 同室・全体

- **同室 2〜10 人での同席観察は、この版ではまだ行っていない。** v0.2.0 から引き継いだ「他者感」「無」の問いは、次の版に回す

## 気づき

### 身体と静けさ

1. **Layer 0 でも、サーボの駆動音は「深呼吸している場所」の静けさと両立しにくい。** v1.0.0 では顔だけに退いた
2. **第三焦点を「机の上の身体」に置く設計は維持しつつ、最初の身体表現は LCD の方が音の面で受け入れやすかった** — 同室観察前の一人での試行として

### 媒体と v0.2.0 からの課題

3. **人間ではないが、Web のように人工的すぎるものでもないロボットは、これまでの課題を解決するための兆しに見える。** v0.2.0 の他者感の薄さや、画面を介した輪郭の薄れ——に対して、第三の存在としての質感がありうる

## v1.1.0 への問い

- 同室 + StackChan（顔のみ呼吸）で、v0.2.0 の「他者感の薄さ」はどう変わるか
- **「人間でも Web でもない」第三焦点** は、他者感・「無」・存在負荷にどう効くか
- 呼吸だけの「無」は、第三焦点（机の上の顔）で輪郭が戻るか、それとも v0.2.0 と同型か
- サーボを再び使うなら、音をどう扱うか（角度を極小にする、使わない、別の身体表現）
- RGB LED で昼夜・フェーズの気配を足すと、存在負荷は下がるか上がるか
