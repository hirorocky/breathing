# journal — バージョンと気づき

探求ループの **観察** と **気づき** を、git のバージョンとセットで残す。

[01-purpose.md](../concept/01-purpose.md) のループ:

```
修正 → 観察 → 気づき → 次のバージョン → 修正 …
```

git では次のように役割を分ける。

| 段階 | git / ファイル | 内容 |
|---|---|---|
| 修正 | 通常コミット | コード・スタイル・振る舞いの変更 |
| 観測版の固定 | `package.json` の version + **annotated tag** `vX.Y.Z` | 観察対象の版を指し示す |
| 観察 | （作業外） | 本番またはローカルで滞在。急がない |
| 気づき | `docs/journal/vX.Y.Z.md` | その版について書く |
| 気づきの固定 | `journal(vX.Y.Z): …` コミット | 気づきをリポジトリに残す |
| 次のバージョン | journal の「次に試す」→ 修正へ | [open-items.md](../tasks/open-items.md) にも転記可 |

**コードのコミットに気づきを詰め込まない。** 実装と観察は別コミットに分け、タグと journal で対応づける。

---

## 1 周の手順

### 1. 修正する

journal の「次に試す」や open-items を参照し、変更を積む。コミットはいつもどおり（小さくてよい）。

### 2. 観測版を切る

観察を始める区切りができたら:

1. `package.json` の `version` を上げる（下記の採番）
2. 変更を `main` に入れ、本番へデプロイ
3. annotated tag を打つ

```bash
git tag -a v0.2.0 -m "観測版: フォーム常時表示・下部レイアウト"
git push origin v0.2.0
```

タグのメッセージは **その版で何を変えたか** の一行でよい。気づきはまだ書かない。

### 3. 観察する

タグが指す版（本番 URL）で、自分や他者の滞在を見る。日数・回数に決まりはない。観察が足りないうちは version を上げない。

### 4. 気づきを書く

`docs/journal/vX.Y.Z.md` を追加する（テンプレートは [v0.1.0.md](./v0.1.0.md) を参照）。

- 存在負荷が下がったか、上がったか
- 想定外の副作用（操作の迷い、緊張、完成感の過剰など）
- 次に試すこと（次の修正の入力）

### 5. 気づきをコミットする

```bash
git add docs/journal/vX.Y.Z.md
git commit -m "$(cat <<'EOF'
journal(v0.2.0): フォーム常時表示で入力の心理的ハードルは下がった

EOF
)"
```

1 行目に版番号。本文はコミットメッセージの残り、または journal ファイルに任せる。

### 6. 次の修正へ

「次に試す」を実装タスクに落とし、手順 1 に戻る。

---

## バージョンの採番

探求用の **観測版番号**。npm 公開や SemVer の厳密さは目的にしない。

| 桁 | 意味 | 上げるタイミング |
|---|---|---|
| `0` | 探求フェーズ | 1 のまま（公開プロダクト化するまで） |
| `X` | 観測サイクル | 気づきを 1 件書き切り、次の観察版を切るとき |
| `Z` | 同一観察版のパッチ | 観察中の軽微な修正。journal は増やさずタグだけ `v0.2.1` 等 |

例: `v0.1.0` を観察 → 気づきを書く → 次のまとまりを実装 → `v0.2.0` をデプロイしてタグ。

---

## ファイル一覧

| ファイル | 役割 |
|---|---|
| `vX.Y.Z.md` | その版についての観察・気づき・次に試す |
| `README.md` | この手順 |

版ごとのファイル名は **タグ名と一致** させる（`v0.2.0` → `v0.2.0.md`）。

---

## git との対応づけ

| 知りたいこと | 見る場所 |
|---|---|
| その版のコード | `git show v0.2.0` |
| その版で何を変えたか | タグメッセージ、または journal の「この版の変更」 |
| その版から何を学んだか | `docs/journal/v0.2.0.md` |
| 気づきを書いた記録 | `journal(v0.2.0):` のコミット |

タグは **観測開始点**、journal は **観測の結果**。セットで 1 サイクルになる。

---

## ルール（短く）

- 気づきなしに `X` を上げない（観察が空のまま進めない）
- タグなしに journal だけ増やさない（どの版の気づきか曖昧になる）
- KPI や滞在時間の数値目標は journal に書かない（質感と副作用を優先）
- 「次に試す」は 1〜3 個に絞る。多すぎると次の修正が散漫になる
