# Journal を書く

存在しやすい空間の探求ループにおける **観察** と **気づき** を `docs/journal/` に残す。

詳細手順・git 対応: [docs/journal/README.md](../../../docs/journal/README.md)

## 原則

1. **journal 依頼では実装を変えない**（`components/` `app/` `worker/` 等）。ユーザーが明示したときだけコードへ手を伸ばす
2. **存在負荷**の観点で書く（下がったか、上がったか、副作用）
3. **KPI・滞在時間・人数目標は書かない**。質感・迷い・寂しさ・操作の重さなど
4. ユーザーの観察文は **要約せず核を残す**。言い換えは最小限
5. 日本語。concept のトーンに合わせ、箇条書きで具体的に

## どのファイルを編集するか

| 状況 | 操作 |
|---|---|
| 版 `vX.Y.Z` の初回まとめ | `docs/journal/vX.Y.Z.md` を新規作成 |
| 同じ版の追加観察 | 既存ファイルの「観察」「気づき」「次に試す」へ統合 |
| 手順・採番の変更 | `docs/journal/README.md`（journal 依頼でなければ触らない） |

ファイル名はタグと一致（`v0.2.0` → `v0.2.0.md`）。タグ未作成の版はメタ行に `（journal 確定後に付与）` と書いてよい。

## 記事テンプレート

```markdown
# vX.Y.Z

- **タグ**: `vX.Y.Z`（未付与ならその旨）
- **本番**: https://breathing-4k7.pages.dev
- **観察期間**: YYYY-MM-DD 〜 …
- **位置づけ**: この版を一言で

## この版の位置づけ

この版が探求ループ上で何を成立させたかを書く。実装の羅列ではなく、観察対象としての意味を書く。

## この版で試したこと

### [領域名]

- 観察に効く変更だけを書く
- 小見出しは `呼吸` `言葉` `気配` `オンライン` `開示と運用` など、版に合わせて調整する

## 観察

### [観察対象]

- **短い見出し。** 具体的な体験

### [観察対象]

- **短い見出し。** 違和感・操作の重さ・寂しさなど

## 気づき

### [気づきのまとまり]

1. **一文の結論。** 理由や含意
2. …

（番号付き。探求の判断に使える形で）

## v次版 への問い

- 実装前の問い・仮説を書く
- まだタスク化しすぎない

## 次に試す（v次版 へ）

- [ ] 1〜3 件に絞る。実装タスクではなく「試すこと」
```

### 追記テンプレート

同じ版への追加は、原則として既存の本文へ統合する。時系列を残したい場合だけ末尾に:

```markdown
## 追記（YYYY-MM-DD）

### 観察

- …

### 気づき

8. **…**（既存番号の続き）

### v次版 への問い（実装前のメモ）

- 仮説・問いのみ。実装はしない
```

追記後は、必要に応じて既存の `## 観察` `## 気づき` `## v次版 への問い` `## 次に試す` に統合し、追記セクションを残すか判断する。

## 書き方のチェックリスト

- [ ] 版番号・観察期間がメタ行にある
- [ ] 「この版の位置づけ」が一言で分かる
- [ ] 観察と気づきが分離している（事実 → 解釈）
- [ ] 気づきは「存在負荷」「他者感」「操作の本数」など探求の語彙で言える
- [ ] 「次に試す」は 3 件以内
- [ ] 実装ファイルを変更していない（依頼が journal のみの場合）

## よくある依頼パターン

**ユーザーが観察を口述しただけ**
→ 該当版の `## 観察` と `## 気づき` に統合。時系列を残す必要があるときだけ `## 追記` を使う。`次に試す` を必要なら更新。

**版の締め・v0.1.0 としてまとめたい**
→ 全文をテンプレートに沿って整理。追記がある場合も本文へ統合し、読みやすさを優先する。git 手順は案内のみ（コミット・タグはユーザー依頼時）。

**journal だけ、実装を変えないで**
→ スキル全体のデフォルト。コード・CSS・docs/concept も触らない（journal 配下のみ）。

## コミット（ユーザーが依頼したときのみ）

気づき確定後、実装コミットとは別に:

```bash
git add docs/journal/vX.Y.Z.md
git commit -m "$(cat <<'EOF'
journal(vX.Y.Z): 一行サマリー

EOF
)"
```

タグ付与は [docs/journal/README.md](../../../docs/journal/README.md) §2 に従う。
